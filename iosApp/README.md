# iosApp

Swift/iOS entry point for RelaxMarket. `iosApp.xcodeproj` was hand-written on Windows (no Xcode
available here) rather than generated by Xcode — it follows JetBrains' standard KMP wizard
template: a single SwiftUI target with one Run Script build phase that builds and embeds the
shared Kotlin framework before linking:

```sh
cd "$SRCROOT/.."
./gradlew :app:embedAndSignAppleFrameworkForXcode
```

No manual Framework Search Paths setup is needed — that Gradle task embeds and signs the
framework directly into the app bundle.

**This project has never been opened in Xcode or compiled on a real Apple toolchain** (Kotlin/
Native targets are disabled on Windows, so `iosArm64`/`iosSimulatorArm64`/`iosX64` compiles are
skipped locally). Treat the first Codemagic build (see `codemagic.yaml` at the repo root) as the
real first compile check — a hand-written `project.pbxproj` is exactly the kind of file that's
easy to get subtly wrong (stale file references, missing build phase wiring) without a real
Xcode to open it in.

Things most likely to need a fix once someone with Xcode looks at this:
- Code signing: `CODE_SIGN_STYLE = Automatic` with an empty `DEVELOPMENT_TEAM`, intended to be
  filled in by Codemagic's `xcode-project use-profiles` or by opening the project in Xcode and
  picking a team once, locally.
- `IPHONEOS_DEPLOYMENT_TARGET = 15.0` — a reasonable default for Compose Multiplatform 1.7.3,
  not verified against this specific app's actual minimum requirements.
- `AppIcon`/`AccentColor` in `Assets.xcassets` are empty placeholders (no image supplied yet).
- The `peekaboo-image-picker`/`peekaboo-ui` (avatar picker/camera) and `qrcode-kotlin` (loyalty
  QR code) iOS actuals under `app/src/iosMain/.../profile/` and `.../util/QrGenerator.kt` are
  the most likely spots for a real compile error — their APIs were verified against the
  libraries' published source/docs but never compiled.
